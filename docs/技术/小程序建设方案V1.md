## 1. 需求梳理与优先级

从会议记录中提取出 7 个核心需求域：

| 优先级 | 需求域 | 会议原话 |
|--------|--------|----------|
| P0 | 小程序报名缴费 | "核心功能就是推送报名窗口及缴费信息" |
| P0 | AI 智能问答 | "AI 无法回答，转人工或用户资料问询" |
| P1 | 用户分层与信息采集 | "提交企业员工、学生类型区别""用户类型（年龄）" |
| P1 | 人工客服兜底 | "人工问答（老师微信、QQ，老师待命）" |
| P1 | 微信生态闭环 | "微信登录（获取微信号、手机号）""AI 推动微信小店" |
| P2 | 五大专区 | "考试专区、学习专区、竞赛专区、活动专区、就业专区" |
| P2 | 课程引流变现 | "NE 课程资源，引流到学习网站，然后缴费学习" |

**关键认知**：甲方要的不是纯客服机器人，而是一个**以 AI 客服为入口、以报名缴费为转化目标**的微信小程序。AI 问答是流量入口，报名缴费是商业闭环。

### 2.2 技术选型

| 层级 | 选型 | 理由 |
|------|------|------|
| 小程序前端 | **原生微信小程序** 或 **Taro** | 需要微信支付、微信登录、手机号获取、微信小店，原生能力调用最顺畅 |
| 后端 API | **FastAPI**（推荐替换 aiohttp） | 已有 Python 基础，FastAPI 生态更成熟，异步、自动文档、校验方便 |
| 数据库 | **PostgreSQL** + **Redis** | PG 存用户/订单/报名表单，Redis 存会话状态、微信 access_token |
| AI 工作流 | **Dify**（已有资产直接复用） | 意图识别 + RAG 检索都已跑通，只需在 Dify 中增加"报名意图"分支 |
| 支付 | **微信支付 JSAPI** | 小程序内统一缴费渠道 |
| 消息推送 | **微信小程序订阅消息** | 报名成功、考试提醒、客服回复通知 |

## 3. 核心功能模块设计

### 3.1 AI 对话模块（已有基础升级）

基于现有 `h3cne_intent_recognizer.py` 和 Dify 工作流，需要**新增两类意图识别**：

| 新增意图类型 | 示例问法 | 处理逻辑 |
|-------------|----------|----------|
| `registration_intent` | "怎么报名""我想考 H3CNE""报名链接" | AI 回答后，主动推送报名入口卡片 |
| `price_inquiry` | "多少钱""企业员工什么价""学生有优惠吗" | 企业员工 **¥1200**，学生 **¥600**；根据用户类型（已识别/待采集）返回对应报价 |

**会话内追加信息采集**：如果用户未登录/未补全资料，AI 在回答后追加引导：

> "为了给你准确的报名信息，请补充：1. 您是企业员工还是学生？ 2. 您的年龄阶段？"

**转人工规则**（触发任一即转接）：
- 意图识别返回 `unknown` 且 RAG 检索无结果
- 用户连续 2 轮输入"转人工""找老师"
- 涉及个人订单、退款、特殊申请等事务性问题

### 3.2 用户与信息采集模块

**微信登录链路**：
1. 小程序调用 `wx.login` 获取 `code`
2. 后端换取 `openid`，建立用户基础记录
3. 需要手机号时触发 `getPhoneNumber` 授权
4. 用户在"我的"页面或 AI 对话中补充：**身份类型**（企业员工/学生）、**年龄**

**用户模型设计**：

```python
class User:
    id: int
    openid: str           # 微信唯一标识
    phone: str | None     # 手机号（授权获取）
    user_type: str | None # "enterprise" | "student"
    age_group: str | None # "under_18" | "18_25" | "25_35" | "above_35"
    created_at: datetime
```

### 3.3 报名缴费模块（P0 核心）

**报名表单设计**（小程序页面）：
- 考试类型：H3CNE / H3CNE-RS+ / H3CSE / H3CIE 等
- 考生信息：姓名、手机号、身份证号（考试用）
- 用户类型：企业员工 / 学生（影响价格）
- 考试券申请：如果是企业员工且批量报名，提供"申请考试卷"入口

**价格差异化**（已确认）：
- 企业员工价：**¥1200**
- 学生价：**¥600**
- 根据 `user_type` 动态显示对应价格
- 价格配置化，后台可调整

**支付流程**：
1. 用户确认报名表单
2. 后端创建订单（状态：待支付）
3. 调用微信支付统一下单，返回小程序调起参数
4. 用户完成支付，回调更新订单状态
5. 推送报名成功通知（订阅消息）

### 3.4 人工客服模块（已确认：3 位老师，个人微信）

**当前现状**：3 位老师均使用**个人微信**接收用户消息，暂无企业微信/自建后台。

**两种形态**：
1. **AI 对话内嵌转人工按钮**：用户点击后，会话标记为"待人工接入"。Phase 1 先生成老师微信二维码供用户扫码添加，Phase 2 再接入消息转发机制
2. **老师待命信息展示**：在"人工问答"专区页直接展示 3 位老师的个人微信二维码、工作时间

**客服分配**：3 位老师轮询分配，或按时段分配（需甲方确认排班方式）。

> **注意**：个人微信无法通过 API 接收/发送消息，Phase 1 只能以"展示二维码 + 用户主动添加"方式实现，后续如需消息直通需迁移至企业微信或自建客服后台。

### 3.5 五大专区（内容展示型，全部从零建设）

> **已确认**：五大专区无现成内容，需甲方从零提供素材（文案、图片、外链地址等）。建议 Phase 1 先搭建框架和占位页，甲方逐步填充内容。

| 专区 | 内容来源 | 交互方式 | 内容建设方 |
|------|----------|----------|------------|
| **考试专区** | 报考信息、考试日历、报名入口 | 跳转报名表单 | 甲方提供考试信息 |
| **学习专区** | NE / 深信服 / IE / P 培训内容 | 引流到外部学习网站 / 微信小店 | 甲方提供学习网站地址及课程资料 |
| **竞赛专区** | 华三杯、NISP 报名信息 | 跳转报名或外链 | 甲方提供竞赛信息 |
| **活动专区** | 运营活动页 | 动态配置 | 甲方提供活动素材 |
| **就业专区** | 合作企业招聘、持证优先岗位 | 信息展示 | 甲方提供招聘信息 |

> 前期可先以静态内容 + 外链方式实现，无需重度开发。但**内容本身需甲方主导产出**，乙方负责技术呈现。

### 3.6 微信小店/课程引流

**AI 推动策略**：
- 当用户询问"怎么学""培训课程""备考资料"时，AI 回答中嵌入学习专区入口或微信小店商品卡片
- 在"学习专区"中设置 NE 课程资源，点击后跳转微信小店或 H5 学习网站

## 4. 与已有资产的衔接

现有底子可以直接复用，不需要推倒重来：

| 已有资产 | 如何复用 |
|----------|----------|
| `h3cne_intent_recognizer.py` | 直接在 Python 后端部署为意图识别服务 API，小程序请求先过此模块 |
| Dify Workflow + RAG 知识库 | 后端转发用户问题到 Dify，返回结果给小程序；报名/价格类问题可在 Dify 中新增分支节点 |
| `dify_spider.py` 测试体系 | 持续回归使用，每次知识库更新后跑一遍测试集 |
| `docs/H3CNE/清洗数据/` | 直接作为 RAG 知识库源，同时部分内容（如报名方式）可同步到"考试专区"展示 |

## 5. 实施阶段建议

建议分 3 个阶段交付，每阶段都有可演示的产出：

### Phase 1：MVP（2-3 周）—— 能对话 + 能报名

- 小程序基础框架搭建
- 微信登录 + 用户信息采集（身份类型：企业员工 / 学生）
- AI 对话接入（复用现有 Dify 工作流）
- 价格查询：企业员工 ¥1200、学生 ¥600（配置化，后台可调）
- 简单报名表单 + 支付（test 环境）

**交付标准**：甲方能在小程序里和 AI 对话问 H3CNE 问题，并完成一笔测试支付

### Phase 2：闭环（2-3 周）—— 能转化 + 能转人工

- 报名缴费正式环境接入
- AI 意图识别新增"报名""价格"分支，支持推送报名卡片
- 人工客服转接：展示 3 位老师个人微信二维码 + 转人工标记（个人微信暂无法 API 接入，用户需主动扫码添加老师）
- 五大专区框架搭建 + 占位页上线（**内容需甲方提供**，Phase 2 先出框架）

**交付标准**：用户从 AI 问答 -> 被引导报名 -> 支付 -> 收到通知，全流程跑通

### Phase 3：运营（持续）—— 能推广 + 能分析

- 微信小店/课程引流接入
- 学习专区外链打通（需甲方提供外部学习网站地址）
- 运营后台（订单管理、用户列表、对话日志查看）
- 数据分析（报名转化率、AI 问题解决率、转人工率）
- 人工客服升级：评估从个人微信迁移至企业微信或自建客服后台，实现消息 API 直通

**交付标准**：甲方能独立运营，每周看数据调整知识库和话术
