# Figma 设计要求文档 V1

> 基于《小程序前端页面设计 V1》整理，供 UI 设计师在 Figma 中输出高保真设计稿

---

## 1. 画布与适配

| 项目 | 要求 |
|------|------|
| 画布尺寸 | 375 x 812 pt（iPhone 14 / 15 基准） |
| 栅格系统 | 左右安全边距 16pt，内容区 343pt |
| 状态栏高度 | 44pt（刘海屏预留） |
| TabBar 高度 | 83pt（含安全区 34pt） |
| 底部安全区 | 34pt（所有底部固定元素需避让） |
| 适配规则 | 按 750rpx 基准，2x 导出 |

## 8. 交互与状态标注

每个页面请在 Figma 中补充 **交互说明注释**（使用 Figma Comment 或独立标注层）：

| 交互点 | 标注内容 |
|--------|----------|
| 按钮点击 | 触发什么动作、跳转到哪个页面 |
| 卡片点击 | 是否有按压态（缩放 0.98，透明度 0.9） |
| 列表滚动 | 是否有下拉刷新、上拉加载 |
| 弹窗出现 | 是否有遮罩层（黑色 40% 透明度）、是否可点击遮罩关闭 |
| 输入框 | 键盘类型（text / number / idcard）、是否自动聚焦 |
| 页面跳转 | 转场方式（底部滑入 / 右侧推入 / 淡入） |
| 加载状态 | Skeleton 展示时长预估、加载失败重试逻辑 |

### 交互动效建议（开发参考）
- 按钮按下：`scale(0.98)` + `opacity 0.9`，150ms ease-out
- 页面转场：右推入 `translateX(100%) → 0`，250ms ease-in-out
- 弹窗出现：遮罩 `opacity 0 → 1`，弹窗体 `translateY(100%) → 0`，200ms ease-out
- 流式输出：文本逐字出现效果，每个字 30ms（设计稿中可展示 3-4 个字的中间态）
- Toast 出现：`opacity 0 → 1` + `translateY(8pt) → 0`，200ms；停留 2s 后淡出

| 空状态插图 | PNG / SVG | 200x200pt | 扁平插画风格，主色点缀 |
