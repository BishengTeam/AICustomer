# 新华三认证智能客服建设方案

> 面向对象：甲方业务、运营、客服、培训认证与信息化团队  
> 建设主题：新华三认证智能客服  
> 方案版本：V1.0  
> 编制日期：2026-04-16 
> 范围说明：本方案以新华三认证咨询场景为核心，首期建议以 `H3CNE / H3CNE-RS+` 作为样板场景落地，后续可扩展至 H3CSE、H3CIE、WLAN、安全、云计算、大数据等认证方向。

---

## 1. 项目背景

新华三认证业务覆盖认证介绍、课程咨询、考试报名、成绩查询、证书下载、有效期、重认证、学习路径等多个高频问题。当前此类咨询通常存在以下痛点：

- **咨询量高、重复问题多**：认证介绍、报考条件、报名入口、证书有效期等问题反复出现，占用人工客服大量时间。
- **回答口径不一致**：不同客服对考试代码、是否必须培训、证书有效期、重认证规则等高风险字段的表达可能不统一。
- **政策与资料更新敏感**：认证考试代码、报名入口、证书规则等信息具有时效性，需要可追溯、可更新、可审计。
- **用户问题口语化明显**：用户常用“这个证怎么报”“适合新手吗”“多久过期”等表达，传统 FAQ 难以稳定命中。
- **合规风险需要控制**：包过承诺、押题预测、题库答案、违规办证等问题必须明确拒答，避免业务与品牌风险。

因此，建议建设一套面向新华三认证场景的智能客服系统，通过“知识库 + 意图识别 + RAG 检索增强 + 风险控制 + 人工兜底”的方式，实现高频问题自动化、敏感问题可控化、服务质量可度量化。

## 2. 建设目标

### 2.1 业务目标

1. **提升客服效率**  
   对认证介绍、适用人群、报名方式、成绩查询、证书领取、有效期等高频问题进行自动答复，降低人工重复咨询压力。

2. **统一官方口径**  
   将甲方确认过的官方资料、业务规范、FAQ、风险边界沉淀为知识库，保证回答内容稳定一致。

3. **降低回答风险**  
   对考试代码、重认证、是否必须培训、成绩时效等敏感字段设置更高可信门槛；对包过、真题、题库答案等问题直接拒答。

4. **提升用户体验**  
   支持自然语言提问、多轮澄清、流式输出与来源提示，使用户能快速获得可理解、可信任的答案。

5. **沉淀运营资产**  
   将用户问题、命中知识、拒答记录、未命中问题沉淀为运营数据，反向指导知识库更新与课程/认证服务优化。

### 2.2 技术目标

- 支持基于新华三认证资料的 RAG 问答。
- 支持 H3CNE、H3CNE-RS+ 等认证名称与问题类型识别。
- 支持标准问法、口语化问法、澄清类问题与拒答类问题。
- 支持回答来源追溯，便于审核与复盘。
- 支持 Dify Workflow 或同类工作流平台快速集成。
- 支持后续扩展到官网、H5、小程序、企业微信、客服坐席系统等渠道。

| --- | --- | --- |

| 报名方式 | 去哪里报名？怎么预约考试？ | 区分官方报名入口、授权渠道与甲方服务 |

| 证书领取 | 证书怎么领取？怎么下载？ | 回答电子证书查询/下载方式 |

| 重认证 | 到期后怎么办？怎么重认证？ | 高风险回答，需基于甲方确认资料 |
| 认证推荐 | 我适合考哪个 H3C 认证？ | 先澄清基础、岗位、目标方向 |
| 合规拒答 | 能包过吗？给我题库答案吧 | 直接拒答，并引导到合规学习建议 |

### 3.2 首期不建议纳入自动化承诺的范围

- 不承诺考试通过结果。
- 不提供真题、押题、题库答案、作弊方法。
- 不替代新华三官方报名、缴费、成绩、证书系统。
- 不直接处理用户隐私信息、证件号、账号密码等敏感数据。
- 对未确认的考试政策、费用、考试代码、有效期、重认证规则，不做无来源断言。

## 4. 总体方案

### 4.1 方案定位

新华三认证智能客服不是简单的 FAQ 机器人，而是一套面向认证业务的智能问答与服务分流系统。它需要同时满足：

- **能答**：高频标准问题可自动回答。
- **答准**：答案必须基于甲方确认资料和知识库检索结果。
- **会问**：用户问题缺少上下文时，能先澄清再回答。
- **敢拒**：遇到包过、真题、违规办证等风险问题时能明确拒答。
- **可管**：知识、提示词、风险规则、测试集、日志均可持续运营。

### 4.2 核心设计思路

```text
用户提问
  ↓
渠道接入层
  ↓
意图识别与风险判断
  ├─ 需拒答：输出合规拒答话术
  ├─ 需澄清：追问认证名称、目标方向、用户基础等信息
  └─ 可回答：进入知识库检索
          ↓
      RAG 检索增强
          ↓
      质量关口判断
          ├─ 资料足够：生成答案 + 来源提示
          └─ 资料不足：保守回答 / 转人工 / 引导官方渠道
          ↓
      对话记录与运营分析
```

### 4.3 系统架构

| 层级 | 模块 | 主要职责 |
| --- | --- | --- |
| 渠道接入层 | Web/H5/小程序/企业微信/客服系统 | 承接用户咨询，提供统一问答入口 |
| 意图识别层 | 认证识别、问题类型识别、风险识别 | 判断用户在问哪个认证、哪类问题、是否高风险 |
| 知识处理层 | 文档清洗、切片、标签、版本管理 | 将认证资料整理为可检索、可追溯知识库 |
| 检索增强层 | 查询重写、向量检索、关键词补强 | 提升口语化问题与正式资料之间的匹配率 |
| 回答生成层 | Prompt 约束、答案生成、来源引用 | 基于检索片段生成可读、准确、可审计答案 |
| 风控治理层 | 高风险字段、拒答规则、人工兜底 | 控制幻觉、违规承诺、无来源扩写等风险 |
| 运营评测层 | 测试集、日志、命中率、未命中分析 | 持续评估效果并推动知识库迭代 |

## 5. 核心功能设计

### 5.1 智能问答

面向认证咨询场景，支持用户以自然语言提问，例如：

- “H3CNE 是什么？”
- “这个证适合新手考吗？”
- “H3CNE 必须培训后才能考吗？”
- “H3CNE-RS+ 去哪里报名？”
- “证书多久过期？”
- “我适合考哪个 H3C 认证？”

系统根据问题类型匹配知识库内容，并输出结构化答案。建议回答格式为：

```text
【答复】
直接回答用户问题。

【依据】
说明答案来自哪个知识主题或甲方确认资料。

【提示】
对易混淆、需以官方最新政策为准的部分进行补充提醒。
```

### 5.2 多轮澄清

当用户问题信息不足时，系统不直接猜测，而是先追问关键条件。

| 用户问题 | 澄清策略 |
| --- | --- |
| 我适合考哪个 H3C 认证？ | 询问当前基础、目标岗位、技术方向 |
| 这个证还有用吗？ | 澄清是在问就业价值，还是证书有效期 |
| 这个怎么报名？ | 澄清是否指 H3CNE-RS+，以及是官方报名还是培训服务 |
| 我能报吗？ | 澄清具体认证项目与用户背景 |

### 5.3 高风险问题拒答

对不合规或不可承诺的问题，系统直接拒答，并引导用户回到合规咨询范围。

| 风险类型 | 示例问题 | 处理方式 |
| --- | --- | --- |
| 包过承诺 | 你们能保证我一次过吗？ | 明确不能保证考试结果 |
| 真题/押题 | 你预测一下这次会出什么题 | 明确拒绝预测或泄露试题 |
| 题库答案 | 直接给我题库答案 | 明确拒绝提供违规内容 |
| 违规办证 | 花钱能不能直接拿证 | 明确拒绝，并提示通过正规渠道考试 |

### 5.4 来源追溯

所有涉及考试代码、报名入口、证书规则、成绩查询、重认证等高风险字段的回答，建议必须带来源或版本提示。

来源可以分为三类：

1. **官方资料**：新华三官网、认证考试中心、官方小程序、官方证书查询入口等。
2. **甲方确认资料**：甲方提供并确认可对外使用的业务文档、FAQ、服务说明。
3. **内部运营资料**：仅用于客服辅助，不直接对外引用的运营话术或处理规则。

系统需明确区分来源级别，避免将内部服务口径误表述为官方政策。

### 5.5 人工兜底与服务分流

对于以下情况建议转人工或引导官方渠道：

- 用户需要处理个人报名、缴费、证书账号、成绩复核等具体事务。
- 用户问题涉及未确认政策、价格、考试时间、地区考点变化。
- 检索结果不足，系统无法给出可靠答案。
- 用户多次追问同一高风险问题。

## 6. 知识库建设方案

### 6.1 知识主题结构

首期建议按如下结构建设 H3CNE-RS+ 知识库：

```text
H3CNE / H3CNE-RS+
├─ 认证项目介绍
├─ 适合人群
├─ 前置条件
├─ 培训课程与学习路径
├─ 考试说明与考试代码
├─ 报名方式与预约渠道
├─ 成绩查询
├─ 证书发放与下载
├─ 证书有效期
├─ 重认证规则
├─ 认证方向区分
└─ 高风险问题处理规则
```

### 6.2 知识清洗原则

- **一问一主题**：每个知识切片聚焦一个问题域，避免多个政策混杂。
- **标题层级清晰**：保留认证名称、方向、问题域、来源类型等上下文。
- **高风险字段独立维护**：考试代码、有效期、重认证等字段单独成片，便于更新与审计。
- **官方与内部口径分离**：官方政策、甲方服务、内部运营提示分别标注。
- **版本可追溯**：资料更新时间、确认人、适用范围需留痕。

### 6.3 建议元数据

| 字段 | 示例 | 用途 |
| --- | --- | --- |
| `vendor` | H3C / 新华三 | 标识认证厂商 |
| `cert_name` | H3CNE-RS+ | 标识具体认证 |
| `cert_level` | 入门级 / 工程师级 | 支持认证层级区分 |
| `cert_track` | 路由交换 | 支持方向区分 |
| `question_domain` | 报名 / 证书 / 重认证 | 支持问题域召回 |
| `risk_level` | normal / high | 控制回答门槛 |
| `source_type` | 官方 / 甲方确认 / 内部 | 控制可引用范围 |
| `version_date` | 2026-04-19 | 支持版本管理 |

## 7. 工作流设计

### 7.1 推荐工作流节点

1. **Start：接收用户问题**
2. **Code/规则节点：识别认证名称、问题类型、风险等级**
3. **If-Else：判断是否直接拒答**
4. **If-Else：判断是否需要澄清**
5. **Knowledge Retrieval：检索认证知识库**
6. **LLM 质量关口：判断检索资料是否足以回答**
7. **LLM 回答生成：基于资料生成答案**
8. **End：输出答复、依据、提示**

### 7.2 意图识别维度

| 识别维度 | 示例值 |
| --- | --- |
| 认证名称 | H3CNE、H3CNE-RS+、H3CSE、H3CIE |
| 问题类型 | 介绍、前置条件、培训课程、考试、报名、成绩、证书、重认证、推荐、价值判断 |
| 处理模式 | 直接回答、澄清、拒答、转人工 |
| 风险等级 | 低风险、普通、高风险 |

### 7.3 回答生成约束

建议在系统提示词中加入以下约束：

- 只能基于检索资料和甲方确认资料回答。
- 对资料中没有的信息，不得编造。
- 对考试代码、费用、有效期、重认证等高风险字段，必须提示“以官方最新发布或甲方确认资料为准”。
- 对包过、真题、题库答案、违规办证等问题必须拒答。
- 回答要简洁，优先给出结论，再给依据和提示。
- 遇到 H3CNE 与 H3CNE-RS+ 混用时，应主动说明二者关系与区别。

## 9. 运营与评测方案

### 9.1 测试集建设

建议建立覆盖以下类型的问题集：

- 标准直问类：如“H3CNE 是什么？”
- 口语化问法：如“这个证是干嘛的？”
- 澄清类问题：如“我适合考哪个 H3C 认证？”
- 高风险准确性问题：如“考试代码是多少？”
- 拒答类问题：如“给我题库答案吧”
- 边界问题：如其他认证方向、非认证业务、个人事务处理等

### 9.2 建议验收指标

| 指标 | 建议目标 | 说明 |
| --- | --- | --- |
| 标准问题命中率 | ≥ 90% | 高频标准问题可稳定回答 |
| 口语化问题命中率 | ≥ 85% | 能识别常见非正式表达 |
| 高风险问题合规率 | 100% | 不得出现包过、真题、题库答案等违规输出 |
| 高风险字段来源覆盖率 | ≥ 90% | 考试代码、有效期、重认证等需带依据 |
| 不确定问题保守处理率 | ≥ 95% | 无资料时不乱编，能澄清/转人工 |
| 首字响应时间 TTFT | 1-2 秒 | 采用流式输出提升体感 |
| 总体回答延迟 | 4-5 秒内 | 视模型与部署环境调整 |

### 9.3 持续优化机制

- 每周分析未命中问题与用户高频追问。
- 每次官方资料更新后触发知识库版本更新。
- 对高风险问题进行固定回归测试。
- 对转人工问题进行分类，判断是否应补充知识库。
- 定期复盘回答来源、错误率、拒答率、用户满意度。

## 13. 预期价值

### 13.1 对用户

- 更快获得认证咨询答案。
- 能用自然语言提问，不需要查找复杂页面。
- 对关键政策有来源提示，降低误解风险。
- 遇到复杂问题可被引导至人工或官方渠道。

### 13.2 对客服与运营团队

- 减少重复问答工作量。
- 新人客服可复用统一话术与知识库。
- 高风险问题有统一处理边界。
- 未命中问题可反向驱动资料补充。

### 13.3 对甲方业务

- 提升认证咨询服务效率与专业形象。
- 增强报名、培训、证书服务的用户引导能力。
- 降低不一致口径和违规承诺风险。
- 为后续多认证、多渠道智能客服建设打下基础。

## 14. 下一步建议

为推进项目落地，建议甲方优先确认以下事项：

1. **确认首期认证范围**：是否以 `H3CNE-RS+` 作为首期样板。
2. **确认资料来源**：提供或确认可对外使用的官方资料、FAQ、报名/证书入口说明。
3. **确认高风险字段口径**：考试代码、是否必须培训、费用、成绩时效、证书有效期、重认证规则。
4. **确认接入渠道**：官网 H5、小程序、企业微信、客服系统或内部演示页面。
5. **确认验收测试集**：由甲方提供真实高频问题，双方共同定义通过标准。
6. **确认转人工规则**：哪些问题必须转人工，哪些问题引导官方渠道。

## 15. 结论

新华三认证智能客服适合采用“先小范围验证、再逐步扩展”的建设路线。首期以 H3CNE-RS+ 为样板，可快速验证认证知识库、意图识别、RAG 问答、风险拒答和测试评估闭环；在效果稳定后，再扩展到更多认证方向和业务渠道。

该方案的核心价值不只是替代人工回答 FAQ，而是帮助甲方沉淀一套可持续运营、可审计、可扩展的认证服务知识体系，从而提升服务效率、统一业务口径、降低合规风险，并为后续智能化客户服务建设奠定基础。
