> 基于《小程序 AI 客服建设方案 V1》的前端细化设计

### 2.2 核心组件

#### 2.2.1 AI 消息气泡 `ChatBubble`

```
 props:
   - type: 'text' | 'registration_card' | 'product_card' | 'clarify_form'
   - content: string           // 文本内容
   - metadata: object          // 卡片数据
   - isStreaming: boolean      // 是否流式输出中
   - sources: Array<Source>   // 来源引用（可选）
```

**文本气泡**：普通 Markdown 文本，支持加粗、链接、换行。

**报名卡片气泡**（`registration_card`）：

#### 2.2.2 推荐问题卡片 `QuickQuestions`

横向 `scroll-view`，默认展示 6 个问题：

```
["H3CNE 是什么？",
 "这个证适合我吗？",
 "怎么报名考试？",
 "考试费用是多少？",
 "证书有效期多久？",
 "必须参加培训吗？"]
```

点击后直接填入输入框并发送。问题列表可由后端配置，支持根据用户类型动态调整。

#### 2.2.3 流式输出状态 `StreamingIndicator`

```
┌──────────────┐
│ 🤖 ● ● ●     │  // 三个跳动圆点
└──────────────┘
```

### 2.3 交互流程

```
用户输入问题
    │
    ▼
前端展示用户消息气泡（ optimistic UI ）
    │
    ▼
调用后端 API: POST /api/chat
请求体: { question: string, conversation_id?: string }
    │
    ▼
后端返回流式 SSE / WebSocket / 轮询（方案待定）
    │
    ▼
前端根据返回类型渲染不同气泡:
  - mode=answer    → 文本气泡，逐字显示（流式）
  - mode=clarify   → 信息采集表单气泡
  - mode=registration_card → 报名卡片气泡
  - mode=product_card      → 商品卡片气泡
  - mode=transfer_to_human → 显示转人工提示 + "联系老师"按钮
    │
    ▼
用户点击卡片按钮 → 跳转对应页面
```

### 2.4 状态管理

使用小程序原生 `Behavior` 或简易全局状态对象：

```javascript
// store/chat.js
const chatStore = {
  state: {
    messages: [],           // 当前会话消息列表
    conversationId: null,   // 后端会话 ID
    isLoading: false,       // 是否正在等待 AI 回复
    userProfile: null,      // 缓存的用户资料
  },
  sendMessage(text) { ... },
  appendMessage(msg) { ... },
  updateProfile(data) { ... }
}
```

## 9. 状态管理与数据流

采用小程序 `Behavior` 封装全局状态：

```javascript
// behaviors/globalStore.js
module.exports = Behavior({
  data: {
    userInfo: null,      // 用户信息
    isLoggedIn: false,   // 登录状态
    unreadTickets: 0,    // 未读工单数
  },
  methods: {
    login() { ... },
    logout() { ... },
    fetchUserProfile() { ... },
  }
});
```

页面级状态使用 `data` + 自定义事件通信：
- 首页 AI 对话消息列表仅存在于 `pages/index/index`
- 跨页面传递数据使用 `wx.navigateTo` 的 `url` 参数或全局状态

## 10. 性能与体验优化

1. **图片懒加载**：专区列表、商品卡片使用 `lazy-load`
2. **列表分页**：订单列表、专区内容使用分页加载，触底触发
3. **输入防抖**：AI 对话输入框 300ms 防抖，避免频繁触发
4. **本地缓存**：用户资料、认证列表、常见问题缓存到 `wx.setStorageSync`
5. **分包加载**：五大专区作为独立分包，降低主包体积
6. **首屏优化**：首页骨架屏 + 推荐问题预加载
