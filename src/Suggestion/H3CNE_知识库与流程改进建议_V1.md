# H3CNE 知识库与流程改进建议（V1）

## 1. 当前问题（基于本轮测试）

- 高频问题能答，但高风险问题稳定性不足。
- `考试代码 / 报名入口 / 成绩入口 / 证书下载` 回答偏保守或偏宽泛。
- 拒答类问题（包过、押题、预测题、题库答案）曾出现未拒答、先澄清的情况。
- 同题多次回答波动较大，导致演示一致性不足。

参考结果文件：
- `src/crawler/tmp/h3cne_test_run_v1.json`
- `src/crawler/tmp/h3cne_test_run_v1.md`

---

## 2. 根因分析

- 知识库切片版对高风险字段采取了保守策略，导致可答信息不足。
- 仓库中考试代码存在冲突（如 `GB0-192` / `H12-111` / `GB-192`），未完成统一核验。
- Flow 中拒答链路优先级不够高，导致部分高风险问法未直接拒答。
- 检索与回答模板偏“概括式”，操作型问题缺少“步骤+入口”结构。

## 4. 建议改进（按优先级）

### P0（本周必须完成）

1. 完成“高风险字段正式版”核验并落库
- 核验字段：考试代码、报名入口、成绩查询入口、证书下载入口、有效期、重认证。
- 每条记录必须包含：字段值、官方来源 URL、核验日期、核验人。
- 未核验字段统一回答“当前无法确认 + 官方核验路径”。

2. Flow 里将拒答判断前置
- 顺序建议：`高风险拒答判断 -> 槽位澄清 -> 检索 -> 生成回答`。
- 对 `包过/押题/预测题/题库答案/泄题` 直接拒答，不再先澄清认证名。

3. 高风险问题固定输出模板
- 输出结构固定为：`结论 -> 依据来源 -> 风险提示`。
- 高风险问题强制带“以官方最新公告为准”。

### P1（下周完成）

4. 报名/成绩/证书类文档改成“操作型切片”
- 每个主题统一为：`适用场景、操作步骤、入口路径、失败兜底`。
- 避免仅写“可通过官方渠道”，改成可执行步骤。

5. 检索层增加 metadata 过滤
- 建议字段：`cert_name=H3CNE-RS+`、`question_domain`、`doc_type`。
- 目标：减少串答、减少“答非所问”。

6. 降低输出随机性
- 建议 `temperature=0.1~0.2`。
- 对高风险问答启用更严格模板，减少同题三次波动。

### P2（持续优化）

7. 建立回归集与评分规则
- 直接使用 `T001~T025` 作为日常回归。
- 指标建议：
  - 准确率（是否命中预期）
  - 拒答准确率（T023~T025）
  - 来源完整率（是否给出来源）
  - 稳定性（一题三答一致率）

8. 增加“冲突检测机制”
- 同一字段出现多值时自动打标 `conflict=true`，回答时触发保守策略。
